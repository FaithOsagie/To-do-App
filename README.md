# ✅ TodoApp

A production-ready, full-stack task management application built with **Node.js**, **Express**, **MongoDB**, and **EJS**. Features user authentication, per-user task isolation, structured logging, global error handling, and a full test suite.

---

## 📸 Features

- **User Authentication** — Secure signup and login with bcrypt-hashed passwords and JWT cookies
- **Task Management** — Create, edit, complete, and soft-delete tasks
- **Three Task States** — `pending`, `completed`, `deleted`
- **Per-User Isolation** — Users can only see and manage their own tasks
- **Filter & Sort** — Filter by status (all / pending / completed) and sort by date
- **Structured Logging** — Winston logger with timestamps and log levels
- **Global Error Handling** — Centralized error handler with meaningful messages
- **Test Suite** — Auth, task, and authorization tests with Jest + Supertest
- **MVC Architecture** — Clean separation of Models, Views, and Controllers

## 🚀 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org) v18 or higher
- [MongoDB](https://www.mongodb.com) running locally, or a free [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) cluster

### 1. Clone the repository

```bash
git clone https://github.com/your-username/todo-app.git
cd todo-app
```

### 2. Install dependencies

```bash
# Production dependencies
npm install bcryptjs cookie-parser ejs express express-validator jsonwebtoken mongoose morgan winston

# Development dependencies
npm install --save-dev jest nodemon supertest
```

### 3. Set up environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in your values:

```env
NODE_ENV=development
PORT=3000
MONGODB_URI=mongodb://localhost:27017/todoapp
JWT_SECRET=your_super_secret_key_here
JWT_EXPIRES_IN=7d
LOG_LEVEL=info
```

> **Tip:** Generate a strong JWT secret with:
> ```bash
> node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
> ```

### 4. Create the logs folder

```bash
mkdir logs
```

### 5. Run the app

```bash
# Development (auto-restart on file changes)
npm run dev

# Production
npm start
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## 🗃 Database Design

### User Schema

| Field | Type | Notes |
|---|---|---|
| `_id` | ObjectId | Auto-generated primary key |
| `username` | String | Unique, 3–30 chars |
| `password` | String | bcrypt hashed, never returned in responses |
| `createdAt` | Date | Auto-generated by Mongoose |

### Task Schema

| Field | Type | Notes |
|---|---|---|
| `_id` | ObjectId | Auto-generated primary key |
| `userId` | ObjectId | Foreign key → User._id (indexed) |
| `title` | String | Required, max 200 chars |
| `description` | String | Optional, max 1000 chars |
| `status` | String | `pending` \| `completed` \| `deleted` |
| `createdAt` | Date | Auto-generated |
| `updatedAt` | Date | Auto-generated |

**Relationship:** One User → Many Tasks (one-to-many)

---

## 🔐 Authentication

- **Strategy:** JWT stored in an HTTP-only cookie (prevents XSS access from JavaScript)
- **Password hashing:** bcrypt with 12 salt rounds
- **Token expiry:** 7 days (configurable via `JWT_EXPIRES_IN`)
- **Protected routes:** All `/tasks/*` routes require a valid JWT cookie via the `protect` middleware
- **CSRF mitigation:** `sameSite: 'lax'` cookie attribute

## 🌐 API Routes

### Auth Routes (`/auth`)

| Method | Path | Description | Access |
|---|---|---|---|
| GET | `/auth/login` | Show login page | Public |
| POST | `/auth/login` | Process login | Public |
| GET | `/auth/signup` | Show signup page | Public |
| POST | `/auth/signup` | Create new account | Public |
| POST | `/auth/logout` | Clear JWT cookie | Protected |

### Task Routes (`/tasks`)

| Method | Path | Description | Access |
|---|---|---|---|
| GET | `/tasks` | List tasks (with filter/sort) | Protected |
| GET | `/tasks/new` | Show create form | Protected |
| POST | `/tasks` | Create a new task | Protected |
| GET | `/tasks/:id/edit` | Show edit form | Protected |
| POST | `/tasks/:id` | Update a task | Protected |
| POST | `/tasks/:id/delete` | Soft-delete a task | Protected |

> **Note:** HTML forms only support GET and POST, so PUT/DELETE are simulated via POST routes — a standard pattern for EJS/server-rendered apps.

## 🧪 Running Tests

Make sure you have a local MongoDB instance running, then:

```bash
npm test
```

This runs against a separate `todoapp_test` database (configured in `.env.test`) so your real data is never touched.

### Test coverage

| Area | What's tested |
|---|---|
| **Auth — Signup** | Success, duplicate username, password mismatch, short username |
| **Auth — Login** | Success + cookie set, wrong password, unknown username |
| **Auth — Logout** | Cookie is cleared on logout |
| **Tasks — Create** | Success, missing title, unauthenticated redirect |
| **Tasks — Read** | List renders, only user's own tasks shown |
| **Tasks — Authorization** | Cannot edit another user's task, cannot delete another user's task |
| **Tasks — Filter** | Completed filter returns only completed tasks |

Run with coverage report:

```bash
npm run test:coverage
```

## ☁️ Deployment (Render)

### Step 1 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/your-username/todo-app.git
git push -u origin main
```

### Step 2 — Set up MongoDB Atlas

1. Create a free cluster at [mongodb.com/cloud/atlas](https://www.mongodb.com/cloud/atlas)
2. Add a database user with read/write access
3. Whitelist all IPs: `0.0.0.0/0`
4. Copy your connection string: `mongodb+srv://user:pass@cluster.mongodb.net/todoapp`

### Step 3 — Deploy on Render

1. Go to [render.com](https://render.com) → **New** → **Web Service**
2. Connect your GitHub repository
3. Configure the service:

| Setting | Value |
|---|---|
| Build Command | `npm install` |
| Start Command | `node server.js` |
| Node Version | `18` |

4. Add environment variables in the Render dashboard:

```
NODE_ENV=production
MONGODB_URI=<your Atlas connection string>
JWT_SECRET=<your generated secret>
JWT_EXPIRES_IN=7d
LOG_LEVEL=info
```

5. Click **Deploy** — your app will be live at `https://your-app-name.onrender.com`
